# Assignment Workflow Walkthrough

You're now ready to work on the first lab assignment.
We have prepared a series of pages that describe the "assignment workflow"; that is, the actions you need to take during each stage of the assignment.
These pages are broken-out by which development environment you plan to use.

In addition to the workflow for working within terminal windows, we describe the workflow for working with [Visual Studio Code](https://code.visualstudio.com/download) and for working with [CLion](https://www.jetbrains.com/clion/download),
the two most popular IDEs for C development.
If you prefer a different IDE, we will try to support you, but please speak with Dr. Bohn.

> ⓘ **Note**
> 
> Your choice of development environment does not have to be "either/or".
> Many students perform some actions in an IDE and other actions in the terminal.
> A common pattern that we've observed is to use the IDE for editing and the terminal for compiling and running the program.

> 💡 **Tip**
> 
> Even if you decide to work exclusively within an IDE, we recommend that you still familiarize yourself with the terminal-based workflow.
> When something doesn't go as expected, repeating the action in the terminal will often provide you with more information that you can use for troubleshooting.

> 🤖 **AI Policy**
> 
> Many IDEs now include AI-assisted coding features.
> Be sure to read the AI Policy in the course's syllabus to understand which in-IDE AI uses are permissible and which are not.
